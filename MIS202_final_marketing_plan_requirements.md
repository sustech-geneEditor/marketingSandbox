## 5. 报告内容范围

报告需要围绕一个可行的 marketing plan 展开。PDF 明确说应与教授讨论哪些内容应该或不应该包含在报告中，但至少应处理以下问题；这些不是穷尽清单，后续 agent 可以根据项目主题补充必要内容。

必须重点覆盖：

- 对营销环境进行更全面的评估。
  - 原文特别说明：相比 proposal presentation，需要更 comprehensive。
  - 不应只是罗列事实或趋势，而应说明这些环境因素如何影响组织的营销努力。
- 明确要达成的 objectives/goals。
  - 目标应服务于营销计划整体逻辑。
  - 后续策略、预算和执行建议应能回扣这些目标。
- 提出实现目标的营销建议。
  - 应体现 customer-driven marketing strategy。
  - 应包含 marketing mix。
  - 需要解释为什么这些建议能达成前述 objectives/goals。
- 提供前三年运营的简单 profit-and-loss statement 和/或 budget。
  - PDF 说明会提供示例。

  - 应与营销目标和执行计划保持一致。

## 10. 评分标准 / Rubric

PDF 对 marketing plan 的评分重点分为三类。后续 agent 应用这些标准反推报告和幻灯片结构。

### 10.1 理解营销环境对组织的深层影响

Above Expectation：

- 全面理解营销环境的不同方面。
- 能说明这些方面如何影响组织的营销努力。

Meets Expectation：

- 对营销环境有充分/基本理解。
- 但可能没有完整说明营销环境如何影响组织的营销努力。

Below Expectation：

- 没有真正分析市场环境，只是重复不同来源中的事实或趋势。
- 没有展示对“营销环境”和“组织营销努力”之间关系的理解。

对后续 agent 的实际要求：

- 不要把市场环境分析写成资料堆砌。
- 每个关键环境因素后都应回答：这对我们的营销目标、目标客户、定位、渠道、定价、推广或预算意味着什么？

### 10.2 理解该情境下的多种可行解决方案

Above Expectation：

- 识别出一组全面、可行且有创造力的选项。
- 清楚理解每个选项的优点和缺点。

Meets Expectation：

- 只识别出少量替代方案，或对未选择的替代方案考虑较浅。

Below Expectation：

- 只提出一个方案，没有考虑其他可能性。
- 面对其他替代方案时，无法捍卫所选方案。

对后续 agent 的实际要求：

- 报告中不要只给最终方案。
- 至少应体现曾考虑过其他战略或战术选项。
- 最终建议应解释为什么被选择，以及为什么其他选项不如它适合。

### 10.3 展示逻辑决策过程

Above Expectation：

- 能连贯、合乎逻辑地呈现和讨论整个决策过程。
- 分析与决策之间有强连接。
- 必要时能使用适当的营销框架和概念。

Meets Expectation：

- 决策过程中遗漏若干步骤，但仍能解释决策理由。
- 分析与决策之间有相对充分的连接。
- 有时能使用适当的营销框架和概念。

Below Expectation：

- 决策步骤缺乏逻辑流。
- 分析与决策之间没有连接。
- 不恰当地使用营销框架/概念，或完全忽略这些框架/概念。

对后续 agent 的实际要求：

- 报告与展示都要形成清晰链条：环境分析 -> 目标 -> 备选方案 -> 选择理由 -> 营销策略 -> 预算/P&L -> 预期结果。
- 不要为了放框架而放框架；框架必须帮助解释判断。
